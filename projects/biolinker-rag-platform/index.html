BioLinker — Medical Paper RAG Analysis Platform

An evidence-oriented RAG service that collects PubMed paper metadata and abstracts, normalizes them into PostgreSQL, and connects vector retrieval with intent-based routing.

Problem Context

Medical literature search often requires both semantic discovery and traceable evidence. The goal was to design a workflow that could retrieve relevant papers, summarize them, and preserve source links to reduce hallucination risk.

What I Did

  • Collected and normalized PubMed paper metadata and abstracts into a PostgreSQL schema.
  • Built ChromaDB-based vector retrieval and organized query flow by intent using LangGraph.
  • Designed a retrieval–summary–evidence connection structure so each answer could be checked against retrieved sources.
  • Implemented FastAPI endpoints and organized logs for reproducible operation and debugging.

Outcome and Portfolio Value

  • Turned a loose paper-search idea into a service-oriented RAG architecture.
  • Practiced evidence-first AI design by separating retrieval, generation, and source-checking responsibilities.
  • Prepared an interview-ready example for AI service architecture, data persistence, and evaluation discussion.
